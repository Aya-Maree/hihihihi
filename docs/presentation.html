CRITERION 2 · CREATIVITY
2 Creative Scenario & Outputs

A guided, context-aware planning conversation

👤 USER
"Birthday party, 25 guests, April 20th, $300 budget at home — 2 vegetarians and some kids."
🤖 AI — Conflict Detection (Step 4)
⚠️ $300 ÷ 25 guests = $12/person — tight. Recommend simplifying menu or increasing budget before I generate the plan.
[Sources: budget_planning_guide · catering_guidelines]
👤 USER
"OK, bump it to $350 and keep it simple."
🤖 AI — Artifacts Ready (Step 7)
✅ Your complete plan: 20 tasks · 28 shopping items · $285.50 estimated · 9 schedule blocks. Ready to download!

What makes this creative:

  • Natural language intake — no rigid forms; the AI extracts structured data from free text
  • Conflict detection before planning — the system refuses to generate a plan with unresolved issues, just like a real consultant
  • Context carries across turns — dietary restrictions, budget updates, and guest count changes are all remembered
  • Conditional branching — dietary flags → dietary KB · children attending → add kids' activities · <7 days → urgency alert
  • Cited responses — every AI answer names the source document, making outputs auditable and trustworthy

CRITERION 3 · STACK & RAG
3 Technical Depth & Execution Quality

Tier 2 tool-enabled system

🧠

RAG Pipeline

  • 23 curated KB documents — birthday, catering, dietary, accessibility, RSVP, schedule templates, vendor ideas
  • sentence-transformers (all-MiniLM-L6-v2) + ChromaDB in-memory vector store
  • Top-5 chunk retrieval with cosine similarity
  • Query enrichment — event type, guest count, dietary needs injected before retrieval
⚙️

Full-Stack Engineering

  • React 18 + Vite + Tailwind — chat, artifact viewer, workflow progress tracker, 3-view renderer
  • Python FastAPI — 12+ REST endpoints, interactive /docs
  • In-memory session state — EventContext JSON persists across all 7 steps and multi-turn chat
  • PDF export via browser print API — no extra library
🔌

Tier 2 — Spoonacular API

  • Search real recipes by event type and dietary needs
  • Ingredient quantities scaled to confirmed guest count
  • Dietary substitution suggestions (vegan, gluten-free, nut-free)
🛡️

Robustness

  • Demo mode — works without API keys via template-based fallback
  • Tested for budget conflicts, dietary edge cases, tight timelines (<7 days)
  • Multi-turn context persistence verified across session lifecycle

CRITERION 4 · OVERALL EFFECTIVENESS
4 Overall Effectiveness of the Final System

Does it actually work? — Yes.

End-to-end user journey:

✓
User describes event in plain English → system extracts all parameters
✓
AI asks targeted follow-up questions — no dead ends
✓
Conflicts flagged and resolved before plan generation
✓
Complete cited plan delivered with real KB sources
✓
3 structured documents generated and downloaded as PDF
✓
Context preserved if user changes budget, guests, or date mid-conversation

What a real user gets: In ~2 minutes of conversation, EventOps AI produces a Task Checklist (~20 tasks across 5 time horizons), a Shopping List (~28 items with budget tracking), and a Day-of Schedule (9 blocks with dependencies) — all grounded in a 23-document knowledge base and exportable as PDF.

Why it goes beyond a basic chatbot: The system enforces a structured 7-step planning process, detects real conflicts before output, cites every source, and produces machine-readable JSON + human-readable Markdown + printable PDF — not just conversational text.
